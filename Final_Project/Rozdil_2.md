# \# 2. Проведення збору і аналізу вимог

# 

# \## 2.1 Обстеження предметної області

# 

# Інформаційна система онлайн-банкінгу призначена для надання клієнтам банку повного спектру фінансових послуг у цифровому середовищі. Система забезпечує безпечний доступ до банківських рахунків, проведення платежів і переказів, управління кредитними та депозитними продуктами, а також формування аналітичних звітів.

# 

# \# Основні бізнес-процеси:



# <ul>

# <li><strong>Реєстрація та автентифікація користувачів:</strong> створення облікових записів, підтвердження особи, відновлення доступу.</li>

# <li><strong>Управління рахунками:</strong> перегляд залишку коштів, історії транзакцій, відкриття або закриття рахунків.</li>

# <li><strong>Платіжні операції:</strong> внутрішні та зовнішні перекази, оплата послуг, встановлення регулярних платежів.</li>

# <li><strong>Кредитування та депозити:</strong> подача заявок на кредити та депозити, контроль виконання умов, облік термінів.</li>

# <li><strong>Аналітика та звітність:</strong> персоналізовані звіти для клієнтів і банку, сповіщення про активність, фінансові рекомендації.</li>

# <li><strong>Безпека та контроль доступу:</strong> багаторівнева автентифікація, моніторинг підозрілих транзакцій, ведення журналу активності.</li>

# </ul>

# \## 2.3 Нефункціональні вимоги

# 

# <ul>

# <li><strong>Надійність:</strong> доступність системи ≥ 99,9%.</li>

# <li><strong>Безпека:</strong> шифрування даних, контроль доступу за ролями, багаторівнева автентифікація.</li>

# <li><strong>Продуктивність:</strong> час відгуку на основні операції ≤ 2 сек.</li>

# <li><strong>Масштабованість:</strong> обробка до 10 000 користувачів одночасно.</li>

# <li><strong>Сумісність:</strong> підтримка сучасних браузерів та мобільних платформ (iOS, Android).</li>

# <li><strong>Зручність використання:</strong> адаптивний та інтуїтивний інтерфейс, багатомовність.</li>

# </ul>

#

# 

# \## 2.4 Обмеження та зовнішні умови

# 

# <ul>

# <li>Інтеграція з банківськими системами та платіжними шлюзами через стандартизовані API.</li>

# <li>Відповідність нормативним вимогам щодо обробки персональних даних (GDPR, локальні закони).</li>

# <li>Використання захищених протоколів передачі даних (HTTPS, TLS).</li>

# <li>Дотримання внутрішніх політик безпеки та аудиту.</li>

# </ul>

# 

# \*\*Результат:\*\* всі вимоги системи оформлено у вигляді \*\*специфікації вимог до програмного забезпечення (SRS)\*\*, що включає функціональні та нефункціональні вимоги, обмеження та зовнішні умови.
